# 项目目的

> 最后更新：2026-05-04

## 一句话

构建公司的 AI 操作系统——让组织运行状态可理解、可协作、可执行、可审计。

## 核心目标

1. **组织可见性**：管理层能在问题恶化前看到真实信号
2. **协作提效**：减少状态同步、周报、会议纪要等事务性工作
3. **员工赋能**：提供能一起讨论、推演、整理的 AI 工作搭档
4. **组织记忆**：每次会议、讨论、决策都沉淀为可检索的公司知识

## MVP 范围（Phase 1：内部 Alpha）

### 必须交付

- [ ] 统一 Agent 入口（模式路由 + 编排）
- [ ] 组织雷达 / 管理层每日简报
- [ ] 项目状态助手
- [ ] 员工每日简报
- [ ] 共事讨论模式
- [ ] 公司知识检索
- [ ] 审批执行队列
- [ ] 权限分层 + 审计日志

### 目标用户

- 管理层（全员）
- 3-5 个团队负责人
- 20-50 名关键员工
- Admin / IT / Security

### 暂不做

- 全员绩效自动评估
- 情绪监控 / 离职倾向预测
- SaaS 多租户
- CEO/HR 默认查看个人原始记忆
- 完整连接器市场

## 成功指标

| 指标 | 目标 |
|------|------|
| 员工每周节省事务性工作时间 | 3-5 小时 |
| 会议行动项捕获率 | > 90% |
| 项目状态同步延迟下降 | 50% |
| 组织风险提前暴露 | 每周 ≥ 3 个 |
| 周报/状态同步时间下降 | 60% |
| 员工隐私边界满意度 | ≥ 8/10 |
| 共事模式每周活跃率 | > 40% |

## 非目标（明确不做）

- 替代现有工具（Slack/Jira/GitHub），我们做的是连接和编排层
- 员工监控系统——不推断情绪，不偷看私人草稿
- 一次性 AI 聊天机器人——核心是持续上下文和组织记忆

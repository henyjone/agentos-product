版本：v2.0  
日期：2026-05-02  
负责人：HGZ

---

## 1. 问题根因

v1.0 的核心错误：AI 只是"润色用户输入的描述"，用户仍然需要手动选 type、填 scope、写描述。这和没有 AI 没有本质区别。

正确的定位：**AI 读取 diff，自动生成符合格式的 commit message，用户只需确认。**

---

## 2. 目标行为

```
用户执行程序
  └── 程序读取暂存区文件列表
  └── 程序读取 git diff --cached（实际改动内容）
  └── AI 分析 diff，生成标准格式 commit message
  └── 展示给用户：
        生成的 message: feat(auth): 新增 JWT 登录接口
        [确认提交]  [手动编辑]  [重新生成]  [取消]
  └── 用户确认 → git commit
```

用户全程不需要手动填写任何内容，除非对 AI 生成结果不满意。

## 4. 新增：`git_utils.get_staged_diff()`

在现有 `git_utils.py` 末尾新增一个函数：

```python
def get_staged_diff(path: str = ".", max_bytes: int = 8000) -> str:
    """返回暂存区的 diff 文本，截断到 max_bytes 避免超出模型上下文。"""
    result = _run_git(["diff", "--cached", "--unified=3"], path)
    if result.returncode != 0 or not result.stdout.strip():
        return ""
    diff = result.stdout
    if len(diff.encode("utf-8")) > max_bytes:
        diff = diff.encode("utf-8")[:max_bytes].decode("utf-8", errors="ignore")
        diff += "\n... (diff truncated)"
    return diff
```

## 5. 重写：`ai_assist.py`

核心变化：输入从"type + scope + 描述"改为"diff 文本"，输出从"润色后的描述"改为"完整的 commit message"。

### System Prompt

```
你是一个 Git commit message 生成助手。
根据提供的 git diff 内容，生成一条符合以下格式的 commit message：

  type(scope): 描述

规则：
1. type 必须是以下之一：feat / fix / refactor / docs / test / chore / perf / revert
2. scope 可选，填受影响的模块名（英文），无明确模块时省略
3. 描述用中文，简洁说明"做了什么"，5-100 字符
4. 只输出 commit message 本身，不要解释，不要加引号

示例输出：
feat(auth): 新增 JWT 登录接口
fix: 修复用户列表分页错误
```

### 接口设计

```python
@dataclass(frozen=True)
class GenerateResult:
    success: bool
    message: str        # 生成的完整 commit message，如 "feat(auth): 新增登录"
    raw_response: str   # AI 原始输出，用于调试
    reason: str = ""    # 失败原因

class CommitMessageGenerator:
    def generate(self, diff: str, staged_files: list[str]) -> GenerateResult:
        """从 diff 生成 commit message。"""
```

### 校验逻辑

AI 输出后用 `types.is_valid_commit_message()` 校验格式。格式不合法时：
- 尝试从输出中提取第一行再校验一次
- 仍不合法则返回 `success=False`，让用户手动编辑

## 6. 重写：`main.py`

### 新流程

```
1. 检测 git 环境
2. 读取暂存区文件列表
   - 为空 → 提示"请先 git add"，退出
3. 读取 staged diff
   - diff 为空 → 提示无内容可分析，退出
4. 调用 AI 生成 commit message
   - 生成中显示等待提示
5. 展示结果：
     暂存区文件 (2):
       src/auth/login.py
       src/auth/middleware.py

     生成的 commit message:
       feat(auth): 新增 JWT 登录接口

     [1] 确认提交
     [2] 手动编辑
     [3] 重新生成
     [4] 取消
6. 用户选择：
   - 确认 → git commit
   - 手动编辑 → 打开编辑器或逐字段引导（type/scope/描述）
   - 重新生成 → 回到步骤 4
   - 取消 → 退出
```

### 关键变化

- 暂存区为空直接退出，不再询问"是否继续"
- AI 不可用时，直接进入手动编辑模式（不是降级为"润色"）
- 手动编辑保留 v1.0 的 type 选择 + 描述输入逻辑（复用现有代码）

## 8. 验收标准

- 暂存区有文件时，AI 自动生成 commit message，用户无需手动填写任何内容
- 生成的 message 格式符合 `type(scope)?: 描述` 规范
- AI 不可用时，程序退化为手动编辑模式，不崩溃
- `--dry-run` 模式下展示生成结果但不执行 commit

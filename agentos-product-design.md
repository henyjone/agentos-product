# AgentOS: AI-Native 公司操作系统产品设计文档

版本：v0.1  
日期：2026-04-30

## 1. 产品定位

AgentOS 不是单纯的员工助手，而是公司的 AI 操作系统。

它连接公司的人、任务、项目、会议、知识、决策和工具，让组织运行状态可理解、可协作、可执行、可审计。

核心定位：

> 一个以工作流自动化为骨架、以员工共事 Agent 为体验核心、以组织记忆为长期复利的公司 AI OS。

首版面向 50-300 人现代 AI 公司，优先服务：

- 管理层
- 团队负责人
- 部分高协作员工
- Admin / IT / Security
- HR 作为有限角色接入

产品路线：先内部系统，后 SaaS 化。

## 2. 核心价值主张

AgentOS 同时解决三类问题。

### 2.1 组织运行问题

公司每天有大量信息分散在 Slack/飞书、Notion、Jira、GitHub、邮箱、会议和文档里。管理层很难及时知道真实状态。

AgentOS 提供：

- 组织雷达
- 项目风险识别
- 跨团队依赖追踪
- 决策闭环
- 管理层每日简报

### 2.2 团队协作问题

团队负责人经常被状态同步、会议纪要、周报、任务跟进消耗。

AgentOS 提供：

- 项目状态自动汇总
- 会议全流程助手
- 阻塞和依赖识别
- 行动项生成
- 工作流审批执行

### 2.3 员工共生问题

员工不只是需要自动化，也需要一个懂上下文、能一起想问题的工作搭档。

AgentOS 提供：

- 个人工作简报
- 共事讨论模式
- 方案推演
- 沟通准备
- 私人思考区
- 从讨论到执行的转化

## 3. 产品原则

1. **公司 AI OS，而不是聊天机器人**  
   聊天只是入口，核心是组织上下文、工作流、权限和执行系统。

2. **Workflow-first**  
   首先让工作流起来：会议、项目、任务、审批、风险、知识检索。

3. **Thinking-partner**  
   Agent 要像共同工作的搭档，能陪员工思考、追问、推演、反驳、整理。

4. **Memory-compounding**  
   每次会议、讨论、决策、项目复盘都沉淀为组织记忆。

5. **Employee-trust by design**  
   不把系统设计成员工监控工具。个人原始记忆默认员工可控。

6. **Suggest + Confirm**  
   Agent 可以读取、总结、草拟、建议；高风险动作必须确认。

7. **Explainable Agent**  
   Agent 要说明它基于什么上下文、进入什么模式、将要执行什么动作。

## 4. 用户角色

### 4.1 管理层

包括 CEO、COO、CTO、业务负责人。

核心需求：

- 今天公司最重要的风险是什么？
- 哪些目标偏离？
- 哪些项目会延期？
- 哪些客户问题在升级？
- 哪些团队负载过高？
- 哪些决策长期没有 owner？

主要功能：

- 组织雷达
- 管理层每日简报
- OKR / 项目 / 客户风险视图
- 跨团队依赖图
- 例外事件提醒

### 4.2 团队负责人

包括 PM、Tech Lead、项目 Owner、部门负责人。

核心需求：

- 项目真实进展是什么？
- 谁被谁卡住？
- 周会前要准备什么？
- 哪些决策需要推进？
- 会后行动项是否落实？
- 怎么少写周报、少同步状态？

主要功能：

- 项目状态助手
- 会议全流程
- 周报生成
- 阻塞识别
- 行动项确认
- 团队知识沉淀

### 4.3 部分关键员工

包括产品、工程、研究、销售、客户成功、运营等高协作岗位。

核心需求：

- 今天最重要的事是什么？
- 哪些会议要准备？
- 哪些任务卡住了别人？
- 谁在等我回复？
- 我能不能和 Agent 一起想清楚一个复杂问题？
- 我的私人思考会不会被管理层看到？

主要功能：

- 个人 Agent
- 每日简报
- 共事讨论模式
- 文档/消息/邮件草拟
- 任务拆解
- 私人记忆管理

### 4.4 Admin / IT / Security

核心需求：

- 哪些工具接入了系统？
- 谁能访问什么数据？
- Agent 做过什么？
- 哪些动作需要审批？
- 是否存在越权或敏感数据泄露？

主要功能：

- 权限策略配置
- 工具连接管理
- 审计日志
- 敏感数据策略
- 访问审批
- 安全事件回溯

### 4.5 HR / People

首版有限接入。

可用能力：

- 员工发展画像
- 能力标签
- 成长目标
- 培训建议
- 组织结构数据

不建议首版直接做：

- 自动绩效判断
- 情绪监控
- 离职倾向预测
- 未授权查看个人原始记忆

## 5. Agent 形态

产品体验上是一个统一 Agent。

用户不需要选择“个人 Agent”“团队 Agent”“管理 Agent”。用户只需要提出问题，系统根据身份、场景、权限和任务自动进入不同模式。

底层模式包括：

1. **Personal Mode**  
   处理个人日程、任务、会议、待办、私人记忆。

2. **Co-Work Mode**  
   和员工一起讨论问题、推演方案、准备沟通、形成判断。

3. **Team Mode**  
   处理项目状态、团队会议、依赖、阻塞、行动项。

4. **Management Mode**  
   处理组织雷达、目标进展、风险、资源冲突。

5. **Knowledge Mode**  
   检索公司知识、会议、文档、历史决策、客户记录。

6. **Execution Mode**  
   调用工具执行任务，但高风险动作必须确认。

7. **Governance Mode**  
   处理权限、审计、安全、敏感数据访问。

Agent 回答时应展示：

- 当前模式
- 使用的数据来源
- 是否涉及敏感信息
- 是否需要确认
- 可执行的下一步

## 6. 核心产品模块

### 6.1 组织雷达

面向管理层。

展示：

- 公司目标进展
- 项目风险
- 客户升级
- 团队负载
- 决策卡点
- 跨团队依赖
- 本周需要管理层介入的事项

核心不是展示所有数据，而是突出：

> 什么变坏了，为什么，谁能处理，下一步是什么。

### 6.2 项目状态助手

面向团队负责人。

能力：

- 从项目工具、会议、聊天、文档、代码仓库汇总状态
- 自动生成项目摘要
- 识别延期风险
- 识别依赖阻塞
- 生成周会 agenda
- 会后生成行动项
- 同步前进入确认队列

### 6.3 个人 Agent

面向员工。

能力：

- 今日工作优先级
- 日程和会议准备
- 待回复事项
- 被阻塞任务
- 阻塞他人的任务
- 文档/消息/邮件草拟
- 个人长期目标和偏好记忆

重点：个人 Agent 不是监控入口，而是员工自己的工作外脑。

### 6.4 共事模式

这是产品的灵魂。

员工可以说：

- “陪我想一下这个方案。”
- “我觉得这个项目有问题，但不知道怎么讲。”
- “帮我准备和老板沟通。”
- “你先反驳我。”
- “把刚才讨论整理成可以发出去的版本。”

共事模式应支持：

- 问题澄清
- 多方案比较
- 反方观点
- 利益相关方分析
- 风险拆解
- 沟通策略
- 决策 memo
- 私人讨论转公开输出

关键边界：

> 讨论过程默认是私人层，只有员工确认后才进入团队或公司上下文。

### 6.5 公司知识与组织记忆

能力：

- 搜索文档、会议、项目、客户、代码、决策
- 回答时提供出处
- 识别重复问题
- 沉淀项目复盘
- 记录重要决策原因
- 支持新人快速了解上下文

长期目标：

> 公司越使用 AgentOS，组织越不容易失忆。

### 6.6 审批执行队列

所有高风险动作进入审批队列。

需要确认的动作包括：

- 发消息
- 发邮件
- 创建正式任务
- 修改项目状态
- 对外沟通
- 访问敏感记忆
- 生成管理报告
- 更新客户记录
- 影响绩效/人事的数据处理

审批记录包含：

- 发起人
- Agent 建议
- 数据来源
- 目标工具
- 影响范围
- 确认人
- 执行结果
- 回滚建议

### 6.7 Admin 与治理中心

能力：

- 工具连接配置
- 角色权限模板
- 数据分层策略
- 敏感词和敏感数据规则
- 审计日志
- Break-glass 访问流程
- 模型和工具调用监控

## 7. 权限与信任设计

数据分为四层。

| 数据类型 | 示例 | 默认可见性 |
| --- | --- | --- |
| 个人原始记忆 | 草稿、偏好、私人反思、未成熟想法 | 员工本人 |
| 工作协作数据 | 任务、项目、会议纪要、PR、公开文档 | 按组织权限 |
| 管理可见数据 | 项目风险、团队负载、目标进展 | 管理层聚合查看 |
| 敏感受限数据 | 薪酬、绩效、健康、法律、财务敏感信息 | 强隔离 |

CEO / HR 可以看：

- 员工发展画像
- 能力标签
- 成长目标
- 公开反馈
- 工作状态摘要
- 组织级风险

CEO / HR 不应默认直接浏览：

- 个人 Agent 原始记忆
- 私人讨论过程
- 未发送草稿
- 情绪和私人反思

但允许受控访问：

- 员工授权
- 合规流程
- 重大业务风险
- 安全事件
- 法律要求

受控访问必须有：

- 访问原因
- 审批人
- 范围
- 时间限制
- 审计记录
- 默认员工通知，法律/安全例外可延迟通知

## 8. 主动性设计

Agent 有一定主动性，但必须克制。

### Level 0：被动响应

用户问，Agent 答。

### Level 1：上下文提醒

例如：

- 会议前准备材料
- deadline 提醒
- 有人等待回复
- 明天客户会前简报

### Level 2：风险触发

例如：

- 项目三天无更新
- 客户问题升级
- 任务依赖阻塞
- 决策没有 owner

### Level 3：共事邀约

例如：

- “你反复修改这个方案，要不要一起梳理？”
- “这个项目风险开始升高，要不要我帮你准备沟通材料？”

首版建议做到：

> Level 1 + 部分 Level 2，谨慎探索 Level 3。

禁止：

- 情绪推断式提醒
- 管理层看到“某员工卡住了”这种私人判断
- 无法解释原因的主动打扰

## 9. 首版核心工作流

### 9.1 管理层每日组织简报

输入：

- OKR
- 项目状态
- 客户记录
- 会议纪要
- 团队任务
- 风险事件

输出：

- 今日公司状态
- 三个最大风险
- 需要管理层介入的事项
- 本周目标偏离
- 建议动作

### 9.2 项目周会

会前：

- 汇总项目进展
- 识别阻塞
- 生成 agenda
- 标出待决策事项

会中：

- 记录讨论
- 捕捉决策
- 捕捉行动项

会后：

- 生成纪要
- 生成任务草稿
- 进入审批队列
- 确认后同步项目工具
- 沉淀到组织记忆

### 9.3 员工每日简报

内容：

- 今日优先级
- 会议准备
- 待回复事项
- 被阻塞任务
- 阻塞他人的任务
- Agent 建议动作

### 9.4 共事讨论到公开输出

过程：

1. 员工在私人空间和 Agent 讨论问题。
2. Agent 帮助澄清、推演、反驳、整理。
3. 员工选择生成公开版本。
4. Agent 生成 memo、消息、任务或会议议题。
5. 员工确认后进入团队上下文。

### 9.5 公司知识检索

用户问：

> 这个客户上次为什么投诉？

Agent 应该：

- 搜索 CRM、会议、工单、聊天记录、文档
- 给出答案
- 标明出处
- 区分事实和推断
- 建议下一步

### 9.6 审批执行

Agent 准备执行动作时：

- 展示动作内容
- 展示数据来源
- 展示影响范围
- 请求确认
- 执行后写入审计日志

## 10. 系统架构

核心模块：

1. **Unified Agent Interface**  
   聊天、工作台、会议页、项目页的统一入口。

2. **Agent Orchestrator**  
   判断当前任务属于哪个模式，并编排工具、上下文和模型。

3. **Workflow Engine**  
   管理会议、项目、审批、提醒、状态同步等流程。

4. **Co-Work / Deliberation Layer**  
   支持共事讨论、方案推演、决策 memo、私人到公开转换。

5. **Context & Memory Layer**  
   管理个人记忆、团队上下文、公司知识和历史决策。

6. **Knowledge Index**  
   索引文档、会议、任务、代码、客户记录和聊天内容。

7. **Permission Policy Engine**  
   判断谁能看什么、Agent 能用什么、哪些动作必须审批。

8. **Tool Connector Registry**  
   连接 Slack、飞书、企微、邮箱、日历、Notion、Jira、Linear、GitHub、CRM 等。

9. **Workflow Approval Queue**  
   统一处理高风险动作确认。

10. **Audit & Governance Layer**  
    记录读取、生成、工具调用、审批、执行、失败和回滚。

## 11. 首版 MVP 范围

必须做：

- 统一 Agent 入口
- 管理层组织简报
- 项目状态助手
- 员工每日简报
- 共事讨论模式
- 公司知识检索
- 审批执行队列
- 权限分层
- 审计日志

暂不做：

- 全员绩效自动评估
- 情绪监控
- 离职倾向预测
- CEO/HR 默认查看个人原始记忆
- 完整 SaaS 多租户
- 完整连接器市场

## 12. 成功指标

首版验证指标：

- 员工每周节省 3-5 小时事务性工作
- 会议行动项捕获率超过 90%
- 项目状态同步延迟下降 50%
- 管理层每周至少发现 3 个提前暴露的组织风险
- 团队负责人周报/状态同步时间下降 60%
- 员工隐私边界满意度不低于 8/10
- 共事模式每周活跃使用率超过 40%

## 13. 产品路线图

### Phase 1：内部 Alpha

对象：

- 管理层
- 3-5 个团队负责人
- 20-50 名关键员工

目标：

- 验证组织雷达、项目状态、个人简报、共事模式是否有真实价值。

### Phase 2：组织工作流闭环

重点：

- 会议到行动项
- 项目到风险
- 风险到管理简报
- 讨论到公开输出
- 审批到执行

### Phase 3：信任与治理强化

重点：

- 权限策略
- 审计
- 敏感数据隔离
- 受控访问
- 员工记忆管理

### Phase 4：SaaS 化

重点：

- 多租户
- 角色模板
- 连接器市场
- 行业工作流模板
- 企业合规
- 管理后台

## 14. 核心产品判断

这个系统最重要的不是“AI 能做多少事”，而是：

> 员工愿不愿意把真实工作过程交给它，管理层能不能从中获得更早、更可信的组织信号。

所以它必须同时做到：

- 对员工有用
- 对团队负责人有用
- 对管理层有用
- 对组织安全可信
- 不让人觉得被监控

一句话总结：

> AgentOS 是公司的 AI 操作系统：用工作流自动化推动组织运转，用共事 Agent 帮员工把问题想清楚，用组织记忆让公司越工作越聪明。
